Project Overview: "Mom's Baking Business"

Project Overview

  • Application Description & Purpose: This web application will be a professional promotional website for a new, home-based baking business. The purpose of the site is to establish a legitimate online presence, showcase the baker's skills and products, and provide a clear method for potential customers to make custom order inquiries.
  • Intended Users: The primary users will be potential customers in the local area (Charlotte, NC) who are looking for custom baked goods for events such as birthdays, holidays, weddings, and corporate functions.
  • Overview of Content: The website will include a homepage with featured items, an "About" page detailing the baker's story, a "Menu" page listing available goods, a "Gallery" of past work, and a "Contact" page with an order inquiry form.

Page Design

The following is a breakdown of each page listed in the site map.

1. Home (index.html)

  • Purpose: To welcome visitors, establish the brand, and direct users to key content areas (Gallery, Menu, Contact).
  • Audience: All users.
  • Content: A brief welcome message, a rotating image gallery (carousel) of featured products, and clear calls-to-action.
  • Data Entry: No.
  • Validations: N/A.
  • Controls: Navigation links, buttons (e.g., "See Our Gallery").
  • Actions: The image gallery will rotate automatically (JavaScript). Buttons will navigate to other pages.

2. About (about.html)

  • Purpose: To build a personal connection and trust with customers by sharing the baker's story and baking philosophy.
  • Audience: All users.
  • Content: Text describing the baker's background, her passion for baking, and her commitment to quality. May include a photo of the baker.
  • Data Entry: No.
  • Validations: N/A.
  • Controls: Navigation links.
  • Actions: None.

3. Menu (menu.html)

  • Purpose: To inform users of the types of products offered (e.g., cakes, cookies, pies) and their descriptions.
  • Audience: All users.
  • Content: A categorized list of baked goods. This content will be loaded from a JSON file.
  • Data Entry: No.
  • Validations: N/A.
  • Controls: Navigation links.
  • Actions: The page will use an AJAX call to fetch and display the menu items from a `menu.json` file.
  • Notes: This fulfills the AJAX requirement.

4. Gallery (gallery.html)

  • Purpose: To visually showcase the quality and artistry of past work, inspiring confidence in potential customers.
  • Audience: All users.
  • Content: A grid of thumbnail images.
  • Data Entry: No.
  • Validations: N/A.
  • Controls: Image thumbnails.
  • Actions: Clicking a thumbnail will open a larger, high-resolution image in a "lightbox" modal overlay (JavaScript/jQuery plugin).
  • Notes: This fulfills the "interactive images" requirement.

5. Contact & Order Inquiry (contact.html)

  • Purpose: To provide a method for users to ask questions or, more importantly, to submit a detailed inquiry for a custom order.
  • Audience: Potential customers.
  • Content: A "Contact & Order Inquiry" HTML form.
  • Data Entry: Yes.
  • Validations: Yes.
    • Name: Required (not empty).
    • Email: Required and must be a valid email format.
    • Date Needed: Required (will use a datepicker).
    • Message: Required (not empty).
  • Controls: Text inputs, email input, textarea, a "Submit" button, and a jQuery-UI Datepicker.
  • Actions: When the user clicks "Submit," JavaScript will validate all required fields. If invalid, it will prevent submission and show error messages. If valid, it will proceed with submission (e.g., using `mailto:` as storage is not allowed).
  • Notes: This fulfills the requirements for a form, JavaScript validation, and a jQuery-UI widget.

Dynamic Functionality on the Website

  • Rotating Image Gallery (Home Page): I will implement a JavaScript/jQuery-based image carousel (e.g., Slick Carousel) on the homepage to display a rotating set of high-quality product photos. (Example: Slick Carousel)
  • AJAX Menu Loading (Menu Page): To make the menu easy for the client to update, the menu items will be stored in a `data/menu.json` file. The `menu.html` page will use a jQuery `$.getJSON()` (AJAX) call to fetch this data and build the menu on the fly.
  • Interactive Photo Gallery (Gallery Page): I will use a jQuery "lightbox" plugin (e.g., Lity, FancyBox) to create an interactive gallery. Users can click a small thumbnail to view a large, high-resolution image in a modal overlay. (Example: Lity Lightbox)
  • jQuery-UI Datepicker (Contact Page): To ensure users select a valid date for their order inquiry, I will use the official jQuery-UI Datepicker widget on the "Date Needed" form field. (Example: jQuery-UI Datepicker)
  • Client-Side Form Validation (Contact Page): Before the inquiry form can be submitted, I will use JavaScript/jQuery to perform client-side validation. This script will check that all required fields are filled out and that the email address is in a valid format, providing real-time feedback to the user if an error is found. This prevents incomplete submissions.
